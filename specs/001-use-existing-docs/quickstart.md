# クイックスタート: 既存ドキュメント準拠の進行

## 目的

既存ドキュメント（`docs/`）を基に、合意事項・矛盾・未決事項を整理し、
新規提案の影響を迅速に判断できる状態にする。

## 手順

1. `docs/00_overview.md` と `docs/README.md` を参照し、
   合意事項と参照元の範囲を確認する。
2. 合意事項/矛盾/未決事項の一覧を `docs/00_overview.md` に集約し、
   `docs/README.md` から参照可能にする。
3. 合意事項の各項目に参照元（ファイル/節）を付与する。
4. ドキュメント間の矛盾はすべて「要判断」として記録する。
5. 未決事項を一覧化し、影響範囲・優先度・担当者（owner）を必ず付与する。
6. 同じ用語に異なる定義がある場合は用語を統合し、単一定義に正規化する。
7. 変更提案がある場合は、既存制約への影響点（追加/変更/削除）を整理する。

## 成果物

- 合意事項一覧（参照元付き）
- 矛盾一覧（ステータス: 要判断）
- 未決事項一覧（影響範囲・優先度・担当者付き）
- 用語の正規化結果
- 変更提案の影響点整理
