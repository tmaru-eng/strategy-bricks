# フェーズ1: データモデル

## エンティティ

### ドキュメント

- **概要**: 合意事項・制約・用語の情報源
- **主要フィールド**:
  - `id`: 一意識別子
  - `path`: ファイルパス
  - `title`: タイトル
  - `lastUpdated`: 最終更新日
  - `sections`: 参照対象セクション一覧
- **関係**:
  - 1つのドキュメントは複数の合意事項を持つ
  - 1つのドキュメントは複数の矛盾/未決事項の参照元になり得る

### 合意事項

- **概要**: 確定済みの制約/契約/用語
- **主要フィールド**:
  - `id`: 一意識別子
  - `category`: 制約/契約/用語
  - `summary`: 要約
  - `term`: 用語（category=用語の場合）
  - `definition`: 定義（category=用語の場合）
  - `sourceRefs`: 参照元（ドキュメントとセクション）
- **検証ルール**:
  - `sourceRefs` は1件以上必須
  - `category` は既定の分類のみ許可
  - `category=用語` の場合は `term` と `definition` を必須

### 矛盾

- **概要**: 文書間の相反記述
- **主要フィールド**:
  - `id`: 一意識別子
  - `description`: 矛盾内容
  - `status`: 要判断
  - `sourceRefs`: 参照元（複数）
- **検証ルール**:
  - `status` は `要判断` のみ許可
  - `sourceRefs` は2件以上推奨

### 未決事項

- **概要**: 決定待ちの論点と仮定
- **主要フィールド**:
  - `id`: 一意識別子
  - `summary`: 要約
  - `impactScope`: 影響範囲
  - `priority`: 高/中/低
  - `owner`: 担当者
  - `sourceRefs`: 参照元
- **検証ルール**:
  - `impactScope` は必須
  - `priority` は高/中/低のいずれか
  - `owner` は必須

## 関係図（テキスト）

- ドキュメント 1..* → 合意事項
- ドキュメント 1..* → 矛盾
- ドキュメント 1..* → 未決事項
